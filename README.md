# MediQ — Intelligent Symptom Diagnostics

MediQ is a machine-learning web app that predicts likely diseases from a set of selected symptoms, then returns a description, precautions, medications, diet, and lifestyle recommendations for the top match. It's built with a scikit-learn Random Forest model and served through a single-file Flask app with a dark, dashboard-style UI.

> ⚠️ **Disclaimer:** MediQ is a learning/demo project, not a medical device. Predictions are generated from a static symptom-disease dataset and must not be used for real diagnosis or treatment decisions. Always consult a qualified healthcare professional.

## Features

- 🔎 Searchable, autocomplete symptom picker with severity indicators
- 🌳 Random Forest classifier trained on 132 symptoms across 41 diseases
- 📊 Confidence gauge plus top-5 predicted diseases with probabilities
- 💊 Per-disease description, precautions, medications, diets, and lifestyle tips
- 🩺 Aggregate severity score based on selected symptoms
- ⚡ Single-file Flask backend, no database required

## Dataset

The model is trained on a public disease–symptom dataset (`archive/Training.csv`): **4,920 records**, **132 binary symptom features**, and **41 disease classes**, supplemented by companion tables for descriptions, precautions, medications, diets, workouts, and symptom severity weights.

## Getting Started

### 1. Clone and install dependencies

```bash
git clone https://github.com/<your-username>/MediQ.git
cd MediQ
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Train the model (generates `artifacts/`)

```bash
python train_model.py
```

This reads the CSVs in `archive/`, trains a `RandomForestClassifier`, and writes `disease_model.pkl`, `symptoms.json`, `knowledge_base.json`, and `model_report.txt` into `artifacts/`.

> The repo already ships with a pre-trained `artifacts/disease_model.pkl`, so this step is optional unless you change the data or want to retrain.

### 3. Run the app

```bash
python app.py
```

Visit **http://127.0.0.1:5001** in your browser, search and select your symptoms, and click submit to see the predicted condition.

## How It Works

1. `train_model.py` cleans and merges all the CSVs in `archive/` into a single knowledge base keyed by disease, and trains a Random Forest on the 132-symptom binary feature matrix.
2. `app.py` loads the trained model bundle and knowledge base at startup, serves the UI, and exposes a `POST /predict` endpoint that accepts a list of symptom IDs and returns the top predicted disease (with confidence, precautions, medications, diets, lifestyle tips) plus the top-5 candidate diseases.
